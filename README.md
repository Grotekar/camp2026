# Лендинг детской выездной смены 2026

Статический лендинг для публикации избранных фотографий после детской выездной церковной смены. Работает без backend, CMS, базы данных, авторизации и сборщика.

## Структура

```text
index.html
style.css
script.js
config.example.js
photos/
.github/workflows/pages.yml
```

Реальный файл `config.js` не хранится в git. Он создаётся локально вручную или автоматически в GitHub Actions перед деплоем.

## Временная заглушка

Сайт по умолчанию закрыт полноэкранным баннером “Страница скоро откроется”. Это удобно, если QR-код или ссылка уже где-то появились, а фотографии ещё не готовы.

Важно: это не авторизация и не настоящая защита. В статическом сайте содержимое всё равно можно найти техническими способами. Заглушка подходит именно как аккуратная временная обложка для обычных посетителей.

Локально заглушка настраивается в `config.js`:

```js
window.LANDING_CONFIG = {
  diskUrl: "https://disk.yandex.ru/i/ВАША-ССЫЛКА",
  isLocked: true
};
```

Чтобы открыть сайт, поставьте:

```js
isLocked: false
```

## Как открыть локально

1. Скопируйте `config.example.js` в `config.js`.
2. В `config.js` замените `diskUrl` на реальную ссылку Яндекс Диска.
3. Оставьте `isLocked: true`, если хотите видеть заглушку, или поставьте `false`, если хотите проверить весь сайт.
4. Откройте `index.html` двойным кликом в браузере.

Интернет нужен только для загрузки шрифта Google Fonts и лёгкой CDN-библиотеки Swiper. Если Swiper не загрузится, галерея покажется обычной аккуратной сеткой.

## Как заменить фотографии

1. Сложите избранные изображения в папку `photos/`.
2. Желательно использовать оптимизированные `.jpg`, `.webp` или `.avif` шириной 1200-1800 px.
3. Откройте `script.js`.
4. В массиве `SITE_CONFIG.photos` замените пути, подписи и alt-тексты:

```js
{ src: "photos/my-photo-01.jpg", alt: "Описание фотографии", caption: "Подпись" }
```

Если фотографий мало, карусель всё равно будет работать. Если список случайно оставить пустым, сайт покажет аккуратную заглушку.

## Ссылка на Яндекс Диск без публикации в git

Переменная окружения помогает не хранить ссылку в репозитории, но не делает её секретной для посетителя сайта. Если кнопка в браузере ведёт на Яндекс Диск, итоговая ссылка всё равно будет видна в исходном коде страницы или сетевых запросах.

Для GitHub Pages ссылка берётся из GitHub Actions secret `YANDEX_DISK_URL` и записывается в `config.js` только во время деплоя.

## Деплой на GitHub Pages

В проект добавлен workflow `.github/workflows/pages.yml`, поэтому лучше использовать деплой через GitHub Actions.

1. Загрузите проект в GitHub-репозиторий.
2. В репозитории откройте **Settings** -> **Pages**.
3. В блоке **Build and deployment** выберите **Source: GitHub Actions**.
4. Откройте **Settings** -> **Secrets and variables** -> **Actions**.
5. Во вкладке **Secrets** добавьте `YANDEX_DISK_URL` со ссылкой на Яндекс Диск.
6. Во вкладке **Variables** добавьте `SITE_LOCKED`.
7. Пока сайт нужно скрывать, поставьте `SITE_LOCKED` равным `true`.
8. Когда сайт можно открыть, поменяйте `SITE_LOCKED` на `false` и заново запустите workflow **Deploy GitHub Pages** во вкладке **Actions**.

Дополнительные переменные для текста заглушки:

```text
LOCK_TITLE
LOCK_TEXT
LOCK_NOTE
```

Если их не задавать, используются стандартные русские тексты.

## Как поменять название смены и даты

Основные тексты находятся в `index.html`. В начале hero-блока оставлен комментарий с местом, где удобно менять название, даты, заголовок и подзаголовок.

## QR-код

После публикации сайта можно создать QR-код на итоговую ссылку страницы и заменить блок `QR` в финальной секции на изображение QR-кода.
